# Word 评估报告审核要点手册

> 本手册覆盖**评估报告**和**评估说明**两种文档类型的审核要点。
> 审核输出遵循三级格式：🔴 硬伤（必须修改） → 🟡 疑问（需核实） → 🔵 优化建议

---

## 准则权威依据

| 准则 | 文号 | 用途 |
|:---|:---|:---|
| **资产评估执业准则——资产评估报告** | 中评协〔2018〕35号 | 报告结构、声明、正文、特别事项、使用限制、附件的**法定要求** |
| **资产评估准则术语** | 中评协〔2020〕31号 | 专业术语的**法定定义**，用于术语准确性核对 |

**关键条文速查**：
- **第十一条**：报告内容 = 标题及文号 + 目录 + 声明 + 摘要 + 正文 + 附件
- **第十二条**：声明 7 项
- **第十四条**：正文 14 项（一~十四）
- **第二十五条**：特别事项 7 类
- **第二十六条**：使用限制 4 项
- **第二十八条**：附件 5 项

## 报告标准章节结构（基于非国资评估报告模板2019版）

```
评估报告
├── 声明（7项）
├── 摘要
├── 正文
│   ├── 一、委托人及其他报告使用人
│   ├── 二、评估目的
│   ├── 三、评估对象和评估范围
│   ├── 四、价值类型及其定义
│   ├── 五、评估基准日
│   ├── 六、评估依据（经济行为/法律法规/准则/权属/取价/会计准则）
│   ├── 七、评估方法（逐项资产评估说明）
│   ├── 八、评估程序实施过程和情况
│   ├── 九、评估假设
│   ├── 十、评估结论
│   ├── 十一、特别事项说明
│   ├── 十二、报告使用限制说明
│   └── 十三、报告日
├── 附件（权属资料/承诺函/资格文件/汇总明细表/差异说明）
```

## 维度一：低级错误检查（Proofreading）

| 检查项 | 说明 | 自动化 |
|:---|:---|:---|
| **错别字/白字** | "被评估单位"写成"背评估单位" | ✅ AI |
| **数字格式不一致** | 同一报告中混用"1,234.56万元"和"1234.56万元" | ✅ 正则 |
| **日期格式不一致** | "2024年3月31日" vs "2024.3.31" vs "2024/3/31" | ✅ 正则 |
| **标点符号** | 中文报告中使用英文逗号、句号；括号半角/全角混用 | ✅ 正则 |
| **重复段落/语句** | 复制粘贴留下的重复内容 | ✅ AI |
| **未替换的模板占位符** | `201*年*月*日`、`**公司`、`XX公司` 等 | ✅ 正则 |
| **编号连续性** | `1. 2. 4.`（跳号）或重复编号 | ✅ 正则 |
| **金额单位不一致** | 同一段落混用"万元"和"元"无说明 | ✅ AI |
| **表格数字精度** | 正文说"1,234.56万元"但表格里是"1,234.57万元" | ⚠️ 半自动 |

## 维度二：描述优化（Writing Quality）

| 检查项 | 说明 |
|:---|:---|
| **过度主观描述** | "最具潜力"、"行业领先"等（第四条：不得有误导性的表述） |
| **企业介绍篇幅** | 模板要求"篇幅不应太大"，是否大段复制企业宣传材料 |
| **表述与收益法假设一致性** | 企业介绍中的行业地位/产能/发展计划 vs 收益法预测的逻辑一致性 |
| **专业术语准确性** | 对照中评协〔2020〕31号术语定义核对，高频清单见下 |
| **语句冗余/啰嗦** | 可精简的长句建议 |
| **评估准则规范用语** | 第五条：提供必要信息使使用人能正确理解评估结论 |

### 高频术语核对清单（来源：中评协〔2020〕31号）

| 报告中常见错误写法 | 准则标准术语 | 术语编号 |
|:---|:---|:---|
| 折现现金流法 / DCF法 | **现金流量折现法** | §44 |
| 成本法（指企业价值） | **资产基础法** | §53 |
| 成本法（指单项资产） | **成本法**（正确） | §1 |
| 自由现金流 / FCF | **股权自由现金流**（FCFE）或应说明口径 | §13 |
| 评估价值 | **评估结论** | §30 |
| 评估报告日 = 签发日 | **可以不同于签发日**（第二十七条） | §56 |
| 市场价值（混淆） | **市场价值** = 自愿买卖方正常公平交易的价值估计数额 | §40 |
| 投资价值（混淆） | **投资价值** = 特定投资者的价值估计数额 | §18 |
| 评估对象/评估范围混用 | **评估对象** = 具体标的；**评估范围** = 表现形态及权利/数量边界 | §26/§27 |
| 无形资产（混淆可辨认/不可辨认） | **可辨认无形资产** vs **不可辨认无形资产（商誉）** | §22 |
| 重置成本 | **复原重置成本**（相同）vs **更新重置成本**（同功能） | §5/§7 |
| 功能性贬值/经济性贬值混用 | **功能性贬值**（技术进步）vs **经济性贬值**（外部条件变化） | §9/§21 |
| 全部权益/部分权益混用 | **股东全部权益价值** vs **股东部分权益价值** | §11/§10 |
| 企业价值/企业整体价值 | **企业整体价值** = 含股东权益+付息债务的全投资口径 | §33 |
| 假设开发法/剩余法 | **假设开发法**（也称剩余法、倒算法） | §16 |
| 多期超额收益法 | **多期超额收益折现法** | §3 |
| 比较法/比较公司法 | **上市公司比较法** 或 **交易案例比较法** | §35/§19 |
| 专业人员/评估师 | **资产评估专业人员**（含非评估师）vs **资产评估师**（持证） | §69/§64 |
| 基准地价法 | **基准地价修正法** | §14 |

## 维度三：内容实质性审核（Substantive Content Review）⭐ 核心维度

> 需要 AI 结合报告实际内容、底稿资产类型、评估方法做实质性的恰当性/充足性/一致性判断。

### 3A. 评估对象和评估范围 — 恰当性审核

| 检查项 | 审核逻辑 | 严重级 |
|:---|:---|:---|
| **评估对象描述准确性** | 评估对象表述是否与经济行为匹配——股权转让→股东全部/部分权益；增资→增资后权益；合并→被合并方净资产等 | 🔴 |
| **评估范围与经济行为一致** | "委托评估对象和评估范围与上述经济行为涉及的评估对象和评估范围一致"——若不一致是否已说明原因？ | 🔴 |
| **资产负债清单完整性** | 报告中列示的资产/负债范围清单 vs 审计报告/申报表——是否有遗漏科目？ | 🔴 |
| **主要资产基本情况** | 模板要求"对整体资产中的主要资产的基本情况进行说明"——大额资产是否有充分描述？ | 🟡 |
| **表外资产披露** | 是否披露了表外资产的类型和数量？ | 🟡 |
| **引用其他报告结论** | 若引用了其他机构报告结论，是否列明了涉及的资产类型、数量和金额？ | 🟡 |
| **合并/母公司口径** | 是否明确说明评估范围是合并报表口径还是母公司口径？ | 🔴 |
| **审计状态** | 是否说明了所用财务报表是"经审计的"还是"未经审计的"？ | 🟡 |

### 3B. 评估依据 — 充足性、匹配性与时效性审核

> **核心规则**：评估依据应由**六大类**构成完整链条。缺一类 = 结构性缺失。

#### 评估依据六大类框架

| 类别 | 检查项 | 审核逻辑 | 严重级 |
|:---|:---|:---|:---|
| **(一) 经济行为依据** | **是否存在该类** | 模板原版缺失此类。应包含：国资/政府批文、董事会/股东会决议、资产重组方案等。法定业务必须有经济行为批文 | 🔴 |
| **(二) 法律法规依据** | **已废止法律检测** | 🔴 **《合同法》已于2021年废止**，应替换为《民法典》 | 🔴 |
| | **跨境法规** | 若涉及境外资产/红筹架构，是否列示了资产所在国家/地区的相关法律法规？ | 🟡 |
| | **最低要求** | 法定业务至少列：《资产评估法》+ 对应的经济行为法律 | 🔴 |
| **(三) 评估准则依据** | **实体性准则匹配** | 扫描评估范围中涉及的资产类型，对照规则表↓检查是否齐全 | 🔴 |
| | **程序性准则链条** | 必须包含：评估程序 + 评估报告 + **评估方法**（中评协〔2019〕35号）+ 委托合同 + 档案 + 职业道德 | 🔴 |
| | **近年新规检测** | 数据资产/核心技术/并购投资价值/司法执行等近年新规是否遗漏？ | 🟡 |
| **(四) 权属依据** | **资产类型对应** | 主要资产是否有对应权属证明文件？ | 🟡 |
| | **抵质押文件** | 若有抵质押资产，是否列示了相关合同和登记证书？ | 🟡 |
| **(五) 取价依据** | **针对性** | 取价依据是否与实际评估方法相关？ | 🟡 |
| | **时效性** | 引用的取价标准/定额是否为最新版本？ | 🟡 |
| | **收益法底层依据** | 若采用收益法，是否列示了商业计划书、可行性研究报告？ | 🟡 |
| **(六) 会计准则依据** | **是否独立列示** | 财报目的评估/PPA/商誉减值等项目建议单独列示 | 🟡 |
| | **跨境会计准则** | 若涉及跨境架构，是否列示了 IFRS / US GAAP？ | 🟡 |

#### AI 审核高频发现模式

1. 🔴 **引用《合同法》** → 已废止，应为《民法典》
2. 🔴 **缺少经济行为依据** → 法定业务尤其需要
3. 🔴 **缺少《资产评估方法》准则** → 程序性准则链条断裂
4. 🟡 **会计准则混入取价依据** → 财报目的项目建议独立

**AI 审核操作逻辑**：
1. 从"三、评估对象和评估范围"中提取涉及的资产类型清单
2. 从"七、评估方法"的子章节标题中提取实际评估的资产类别
3. 对照上述四类规则表，检查"六、评估依据 (二/三)评估准则依据"是否齐全
4. 检查程序性准则链条是否完整
5. 输出：🔴 应列未列的实体准则 / 🟡 程序性准则缺失 / 🔵 列了但实际未涉及的准则

### 3C. 特别事项说明 — 恰当性与充分性审核

> **特别事项的性质是免责声明——既不能啰嗦冗余，也不能遗漏关键风险。**
> 准则第二十五条列了 7 类：权属瑕疵/关键资料/未决事项/专家工作/期后事项/程序受限/其他。

#### 必备标准免责项清单

| 编号 | 标准项 | 适用条件 |
|:---|:---|:---|
| 1 | 产权状况说明 | **所有项目** |
| 2 | 抵押/质押/担保/诉讼/承诺事项 | **所有项目** — 无则说明"截至基准日不存在" |
| 3 | 评估结论不等于可实现价格 | **所有项目** |
| 4 | 假设前提变化免责 | **所有项目** |
| 5 | 不对委托方提供资料的真实性负责 | **所有项目** |
| 6 | 瑕疵事项免责 | **所有项目** |

#### 条件触发项

| 编号 | 条件触发项 | 触发条件 |
|:---|:---|:---|
| 7 | 设备技术参数未做技术检测 | 评估范围有机器设备 |
| 8 | 建构筑物隐蔽工程未做技术检测 | 评估范围有房屋建筑物 |
| 9 | 评估程序受限及弥补措施 | 有任何评估程序受限情况 |
| 10 | 期后重大事项 | **所有项目** — 必须说明是否存在 |
| 11 | 账外无形资产排除 | **仅资产基础法** — 收益法/市场法应删除此条 |
| 12 | 未考虑资产评估增减额税收影响 | 资产基础法，且需写明具体哪些科目 |
| 13 | 部分股权 vs 全部股权 | 经济行为为部分股权转让时 |
| 14 | 流动性因素说明 | **仅市场法**结论时需说明 |
| 15 | 部分资产评估是否含税 | 评估对象为单项/部分资产时 |
| 16 | 评估盘盈/盘亏处理 | 清查发现盘盈盘亏时 |

#### 审核要点

| 检查项 | 审核逻辑 | 严重级 |
|:---|:---|:---|
| **必备项是否齐全** | 对照6条必备项检查是否遗漏 | 🔴 |
| **条件触发项是否匹配** | 根据评估范围/方法/资产类型检查条件触发项是否已纳入 | 🔴 |
| **描述恰当性** | 每条措辞是否准确反映实际情况？是否模板原文照抄但与项目不符？ | 🟡 |
| **啰嗦/冗余检测** | 是否有与项目无关的模板残留？（如港口煤炭、香港子公司、煤矿井下建筑物等特定场景模板文本） | 🟡 |
| **期后事项** | 是否说明了基准日后至报告日的重大事项及影响？ | 🔴 |
| **产权瑕疵** | 若有产权不清晰的资产，是否已在特别事项中披露？ | 🔴 |

### 3D. 估值技术路线与参数合理性审核

> 审核**报告文本中对方法和参数的描述/论证是否合理充分**，而非 Excel 计算正确性。

#### 评估方法选择的合理性

| 检查项 | 审核逻辑 | 严重级 |
|:---|:---|:---|
| **方法适用性论证** | 选用的方法是否适用于该企业？例如：初创/连续亏损企业采用收益法依据是否充分？ | 🔴 |
| **未选方法的排除理由** | 第二十一条要求仅用一种方法时必须说明原因。排除理由是否合理？是否"模板化排除"？ | 🔴 |
| **多方法差异分析** | 两种以上方法时，差异原因分析是否合理？最终选取理由是否充分？ | 🔴 |
| **方法与价值类型一致** | 方法选择是否与价值类型匹配？（投资价值→收益法；清算价值不应用持续经营假设） | 🔴 |

#### 关键参数与假设的合理性

| 检查项 | 审核逻辑 | 严重级 |
|:---|:---|:---|
| **折现率参数描述** | WACC/CAPM 各项参数取值是否有描述和依据说明？ | 🔴 |
| **核心假设支持** | 预测假设是否有市场/历史数据支持？是否仅写"管理层预测"无评估师独立判断？ | 🟡 |
| **现金流↔折现率口径一致** | FCFF↔WACC、FCFE↔Ke；税前↔税前折现率 | 🔴 |
| **评估假设逻辑自洽** | 目的、价值类型、假设、参数、结论在逻辑上是否一致？ | 🔴 |
| **永续期假设描述** | 永续增长率、CapEx 与 D&A 关系等是否有说明？ | 🟡 |
| **资产剩余寿命** | 无形/固定资产的剩余经济寿命设定是否有依据？ | 🟡 |

### 3E. 评估程序合规与底稿支持审核

> 审核"八、评估程序实施过程和情况"章节的描述是否充分。

| 检查项 | 审核逻辑 | 严重级 |
|:---|:---|:---|
| **现场调查描述** | 是否说明了现场调查实施情况？有无具体时间/地点/参与人员？ | 🟡 |
| **核查验证程序** | 对重要资产是否说明了具体核查验证措施？ | 🟡 |
| **程序受限披露** | 程序受限是否在"八、评估程序"和"十一、特别事项"中均披露？（第七条） | 🔴 |
| **替代程序说明** | 程序受限时是否说明了弥补措施？措施是否合理？ | 🔴 |
| **重大影响判断** | 第七条：程序受限对结论产生重大影响或无法判断→禁止出具报告 | 🔴 |
| **资料收集描述** | 是否说明了评估资料收集整理的主要内容？ | 🟡 |

### 3F. 资产权属与期后事项专项审核

> 这两个事项是引发重大执业风险的高频领域。与 3C 有交叉但视角不同——这里审核报告正文中的实质性描述质量。

#### 资产权属核查与瑕疵披露

| 检查项 | 审核逻辑 | 严重级 |
|:---|:---|:---|
| **重要资产权属清晰度** | 土地、房产、核心知识产权等的权属证明是否描述清楚？ | 🔴 |
| **权属瑕疵处理** | 权属不完备资产的处理方式是否合理？（设定产权假设/承诺函等） | 🔴 |
| **瑕疵风险提示** | 权属瑕疵是否在"三、评估范围"和"十一、特别事项"中均披露？有无仅一处提及的遗漏？ | 🔴 |
| **抵质押/查封状态** | 是否披露了资产的抵押、质押、查封等限制情况？ | 🟡 |

#### 期后事项的关注与处理

| 检查项 | 审核逻辑 | 严重级 |
|:---|:---|:---|
| **期后重大事项识别** | 基准日至报告日间是否有重大诉讼/汇率波动/政策变化/突发事件？ | 🔴 |
| **处理方式审核** | ①评估处理型（如税费政策调整）→是否合理？②评估披露型→是否充分披露？ | 🔴 |
| **基准日-报告日时间跨度** | 超过3个月时是否特别关注了期后事项？ | 🟡 |
| **多处描述一致性** | 期后事项在"八、评估程序"、"十一、特别事项"、"十二、使用限制"中的描述是否一致？ | 🟡 |

## 维度五：模板合规检查（Template Compliance）

> 以下检查项均有准则条文级依据（中评协〔2018〕35号）。

| 检查项 | 准则依据 | 审核逻辑 |
|:---|:---|:---|
| **报告结构完整性** | **第十一条** | 必须包含：标题及文号、目录、声明、摘要、正文、附件 |
| **正文14项完整性** | **第十四条** | 正文必须包含（一）~（十四）共14项 |
| **声明7项完整性** | **第十二条** | 声明必须包含7项标准内容 |
| **使用限制4项** | **第二十六条** | 使用范围、违规免责、使用人限制、结论≠可实现价格 |
| **附件5项完整性** | **第二十八条** | 权属资料、承诺函、资格文件、汇总明细表、差异说明 |
| **评估目的唯一** | **第十六条** | 报告载明的评估目的应当唯一 |
| **价值类型有理由和定义** | **第十八条** | 应说明选择价值类型的理由并明确定义 |
| **评估方法选择理由** | **第二十一条** | 仅用一种方法时必须说明其他方法不适用的原因 |
| **有效期表述** | **第十条** | 评估基准日与经济行为实现日不超过一年 |
| **签名盖章** | **第八条** | 至少两名评估专业人员签名 + 机构印章 |
| **中文撰写** | **第九条** | 必须使用中文；外币需注明基准日汇率 |
| **基准日一致性** | **第十九条** | 全文中基准日应一致 |
| **被评估单位名称一致性** | — | 全文中公司名称应一致 |

## 维度六：与 Excel 底稿勾稽审核（Word-Excel Cross-check）

> 报告与底稿之间的数据/方法口径一致性。

| 检查项 | 说明 | 示例 |
|:---|:---|:---|
| **评估方法名称一致** | 报告写的方法类型 vs 底稿实际计算口径 | 报告写 FCFF，底稿实际是 FCFE |
| **评估结论数值一致** | 报告中的结论金额 vs 底稿最终结果表 | 增值率不一致 |
| **关键参数一致** | WACC、永续增长率、折现率、资本结构等 | WACC 数值不同 |
| **基准日数据一致** | 报告披露的资产/负债/权益账面值 vs 底稿 | 资产总额不一致 |
| **收入/利润预测一致** | 报告中的预测数据 vs 底稿预测表 | — |
| **评估方法选择理由** | 报告中说的方法 vs 底稿中实际使用的方法 | 报告说两种方法但底稿只有一种 |
| **税率一致** | 报告中披露的税率 vs 底稿税率假设 | — |
| **折旧/摊销方法** | 报告中的会计政策描述 vs 底稿折旧假设 | — |

## 维度七：企业信息核对（iFinD Company Data Verification）

> 报告中涉及的公司详细信息需要与第三方数据源核对。

### 工作流

```
Step 1: AI 从 Word 报告中提取所有涉及的公司名称
        ↓
Step 2: AI 将公司名称填入 iFinD 批量查询模板（templates/ifind_batch_query_template.xlsx）
        → 第二行起，每行一个公司全称
        ↓
Step 3: 用户在 iFinD 中导入模板，批量导出企业工商信息
        ↓
Step 4: 用户将导出的 Excel 交回 AI
        ↓
Step 5: AI 逐项核对导出数据 vs Word 报告中的描述
```

### 核对字段清单

| Word 报告描述 | 对应 iFinD 导出字段 | 核对重点 |
|:---|:---|:---|
| 公司名称 | C列(公司名称) + D列(曾用名) | 是否用了曾用名但写的现用名 |
| 注册资本 | O列(注册资本) + P列(币种) | 金额和币种 |
| 成立日期 | R列(成立日期) | 与报告中"成立于XX年"对照 |
| 法定代表人 | N列(法人代表) | 是否为最新法代 |
| 经营范围 | AO列(经营范围) | 主营业务是否属于经营范围 |
| 公司类型 | J列(公司类型) + K列(企业性质) | 企业类型描述 |
| 注册地址 | AB列(企业地址) | 注册地址 |
| 统一社会信用代码 | X列(统一社会信用代码) | — |
| 经营状态 | E列(经营状态) | 非"存续"需特别提示 |
| 行业分类 | AI~AM列(国民行业) | 行业描述准确性 |

## 审核执行三阶段流程

```
阶段一：文本质量审核
  → 低级错误扫描（维度一）
  → 描述优化建议（维度二）
  → 模板合规检查（维度五）
  → 错误引用检查（维度四）
  → 内容实质性审核（维度三 3A-3F）

阶段二：数据勾稽审核（需要底稿）
  → 提取报告中所有数值/参数
  → 提取底稿关键结论/参数
  → 交叉比对（维度六）

阶段三：企业信息核对（需要 iFinD 导出）
  → 提取报告中涉及的公司名称
  → 生成 iFinD 批量查询 Excel
  → ⏸ 等待用户导出 iFinD 数据
  → 核对导出数据 vs 报告描述（维度七）
```
